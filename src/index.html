📡 무선 연결 방법

            

              

                
⭐ 권장: 5555 포트 고정 (1회 세팅)

                
    
                  
  1. USB로 디바이스 연결 (또는 무선이 이미 연결된 상태)
    2. 
                  
  2. 상단 무선 연결 → 📌 5555 포트로 고정 클릭
    3. 
                  
  3. 이후 앱을 켤 때마다 자동 재연결됨 (VPN/네트워크 바뀌어도 OK)
    4. 
                

                
💡 폰 재부팅 시 무선 디버깅 포트 초기화 → USB로 한 번 다시 5555 고정 필요

              

              

                
대체: 수동 페어링 (USB 없이 첫 연결)

                
    
                  
  1. 폰: 설정 → 개발자 옵션 → 무선 디버깅 ON
    2. 
                  
  2. "페어링 코드로 기기 페어링" 터치 → IP:Port / 6자리 코드 표시
    3. 
                  
  3. 앱: 무선 연결 모달에서 페어링 IP·코드 입력 → 페어링
    4. 
                  
  4. 같은 화면 하단 연결용 IP:Port 입력 → 연결
    5. 
                

              

            

          

          

            
🆕 최근 추가된 기능 사용 방법

            

              

                
신규Jira 버그 티켓 자동 생성

                
    
                  
  1. 좌측 메뉴 → Mirror & Inspect 진입
    2. 
                  
  2. 우측 상단의 + Create Jira Bug Ticket 버튼 클릭
    3. 
                  
  3. 최초 1회: ⚙️ 아이콘으로 Atlassian 이메일 + API Token 입력
                    (토큰 발급: id.atlassian.com/manage-profile/security/api-tokens)
    4. 
                  
  4. 제목 / Bug Category / Linked Item / Labels 입력 후 티켓 생성
    5. 
                  
  5. 현재 디바이스의 OVERDARE 앱 정보 + 스크린샷 + Meta.log + LogHandler 로그가 자동 첨부됨
    6. 
                

              


              

                
신규Jira 티켓 Reopen

                
    
                  
  1. Mirror & Inspect 우측 상단의 빨간 Reopen 버튼 클릭
    2. 
                  
  2. 리오픈할 Issue Key 입력 (예: QA-9162)
    3. 
                  
  3. 댓글 본문에 앱 정보 + 안내 문구가 자동 prefill (수정 가능)
    4. 
                  
  4. 스크린샷 / 로그 자동 첨부 (필요 없는 항목은 ✕ 로 제외 가능)
    5. 
                  
  5. Reopen 클릭 → 첨부 업로드 → 댓글 작성 → 상태가 Reopened 로 자동 전이
    6. 
                

              


              

                
신규BVT 자동화 테스트

                
    
                  
  1. 좌측 자동화 폴더 → 자동화 테스트 진입
    2. 
                  
  2. 디바이스 선택 (무선 디바이스는 config.py 에 자동 등록됨)
    3. 
                  
  3. 실행할 시나리오 체크 (시나리오 목록은 main.py 에서 자동 파싱)
    4. 
                  
  4. 옵션 선택 (Slack 알림 / 비디오 녹화 / 미러링 등)
    5. 
                  
  5. 실행 클릭 → 콘솔에서 실시간 결과 확인, 종료 시 자동 정리
    6. 
                

              


              

                
신규웹뷰 (Jira / Confluence / Sable 등) 편의 기능

                
    
                  
  • 마우스 옆 버튼 또는 Alt + ←/→ 로 페이지/패널 뒤로/앞으로
    • 
                  
  • Ctrl + +/- /0 또는 Ctrl + 휠 로 페이지 확대/축소 (앱 재실행해도 유지)
    • 
                  
  • Confluence 에서 Jira 링크 클릭 시 → Jira 패널로 자동 이동
    • 
                  
  • 각 웹뷰 카테고리는 마지막 본 URL 을 기억
    • 
                

              


              

                
신규🎨 Figma Link Generator (v1.1.0~v1.1.2)

                
    
                  
  1. 좌측 자동화 폴더 → 🎨 Figma Link 진입
    2. 
                  
  2. 최초 1회: ⚙️ Token 으로 Figma Personal Access Token 입력
                    (발급: figma.com/settings/tokens · 권한: file_content:read, file_comments:read, file_metadata:read)
    3. 
                  
  3. 파일 URL 붙여넣고 불러오기 → 페이지/프레임/섹션 트리 표시
    4. 
                  
  4. 좌측 Pages 사이드바에서 필요한 페이지만 켜기 (Cover 같은 거 끄기)
    5. 
                  
  5. 체크박스 다중 선택 → 📋 다음 묶음 복사 로 30개씩 분할 복사 또는 📋 전체 복사
    6. 
                  
  6. 노드 옆 💬 N 뱃지 클릭 시 코멘트 미리보기 (코멘트 옵션 ON 시)
    7. 
                

                
⚠ 같은 파일 6시간 내 캐시 · 분당 5회 호출 제한 (Figma rate limit 보호)

              


              

                
신규Install App — OVERDARE 자동 설치

                

                  Firebase App Distribution 의 dev / shipping 빌드를 선택해 연결된 디바이스에 자동 설치하는 패널입니다.
                

                
    
                  
  1. 좌측 메뉴 → 모니터링 → Install App 진입
    2. 
                  
  2. 최초 1회: 우측 상단 ⚙️ → GitHub Enterprise PAT 입력 후 저장
                    
      
                      
    • 발급: github.krafton.com/settings/tokens
      • 
                      
    • Generate new token (classic) → Scopes: repo, workflow 체크
      • 
                    
    
                  
    3. 
                  
  3. 설치 대상 영역에서 디바이스 토글로 다중 선택 (전체 버튼으로 일괄 토글)
    4. 
                  
  4. DEV / SHIPPING 컬럼의 ↻ 버튼으로 빌드 목록 새로고침 (러너 실행 포함 30~90초 소요)
    5. 
                  
  5. 원하는 빌드 카드의 설치 버튼 클릭
                    
      
                      
    • APK 1회 다운로드 → 선택된 모든 디바이스에 병렬로 클린 설치 (기존 앱 삭제 후 설치)
      • 
                      
    • 완료 시 카드 옆에 ✓ 디바이스명 표시 (현재 어떤 디바이스에 깔린 빌드인지 한눈에 확인)
      • 
                    
    
                  
    6. 
                

                
💾 빌드 목록은 캐시되어 앱 재시작 후에도 유지됩니다.

              


              

                
신규설정 (다크/라이트 모드 · 알림)
